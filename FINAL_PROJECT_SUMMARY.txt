================================================================================
                        KEY ARCHITECTURAL PATTERNS
================================================================================

1. VALIDATION PATTERN
   Returns: (is_valid: bool, error_message: str | None)
   Benefits: Clear success/failure, informative error messages

2. ERROR HANDLING PATTERN
   Returns: (result: Any, error_message: str | None)
   Benefits: Easy to chain operations, clear error propagation

3. MODULAR DESIGN
   • Main app: User interface and loops
   • Logic module: Pure functions without I/O
   • Benefits: Testable, reusable, maintainable

4. TEST STRUCTURE
   • Separate test file per task
   • Test class per function
   • Multiple test methods per function
   • Integration tests at end

5. DOCUMENTATION
   • Header with developer info and date
   • Function docstrings with parameters/returns
   • Test descriptions
   • README with examples
